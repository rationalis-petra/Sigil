A Brief Intro to Seuth

Seuth is a second programming language embedded within Glyph, but it is a very different type of language: While Glyph is mostly about buliding functions which perform some kind of computation, Seuth is all about solving equations and answering questions. While programming, your Seuth code will have access to all your Glyph code and vice-versa.

We can build a simple seith query with the below code: 

(main : IO Unit)
(main ≜ do
  (let (x ≜ seuth.query (∃ A. 'A + 2 ≡ 3))) ;; TODO: figure out embedding syntax
  (print (head x)))

The ' on 'A is tells Seuth that A is a variable, and it needs to deduce the variable's value from the information provided. The output of a seuth query is a stream of all possible solutions. In this case, the equation has only one solution (A = 1), so this code will print '1'. This was just a simple query: the full Seuth language has many more features for directing how to search for a solution, but all Seuth programs are structured as questions of this form.

Uses

Seuth is considered an advanced part of the Glyph language, and understanding it properly is generally not necessary when writing programs. It is nevertheless useful in several areas:

Typeclasses
Typeclasses are a features of many programming languages, Glyph included, however they are implemented in Glyph as Seuth queries. To learn more about typeclasses, you can read the ext tutorial in this series.
Metaprogramming
A metaprogramm is a program which writes another program. Seuth's query system is uniquely good at helping us write metaprograms: we can specify a set of properties we want our program to have, and let Seuth do the work of finding a program which matches those properties.
Specific Domains
Seuth is a type of language called a logic programming langugage. There are many other logic programming languges, but the most relevant ones are Prolog, which is implemented in the Swi-Prolog system, and λProlog, which is notably implemented in the Teyjus and Makam systems. These languages have proven to be generally useful, but also particularly useful for specific domains, like Classical AI, implementing programming languages and constraint logic programming.
