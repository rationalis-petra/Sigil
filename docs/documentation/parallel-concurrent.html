Parallelism and Concurrency in sigil

sigil / Documentation / Parallel & Concurrent

Overview

sigil has an generally unopinionated approach to concurrency, and encourages the use of whatever techniques work best for the particular use-case. Some questions you might want to ask yourself when choosing an approach is:

  • Do you need lots of concurrency, i.e. will there be lots of I/O happening simultaneously?
  • Will you have lots of work running in parallel, or are there only a small number of parallel workloads?
  • Do parallel units need to communicate, or are they isolated?
  • Do you just need parallelism for performance improvements?

inc & dec - mutual processes

  • inc: sends n + 1 to dec
  • dec: sends n + 1 to inc
(induct Tree
  (leaf : Tree)
  (branch  : ℕ → Tree → Tree))

(print-tree : Behaviour Tree)
(print-tree ≜ specify ⧺ λmatch
  (leaf       → pure ())
  (branch n l r → do
    (print "{n}, ")
    (spawn (print-tree l))
    (spawn (print-tree r))))

(make-tree : Array ℕ → Tree)
(make-tree arr ≜
  if (even? ⧺ len arr)
     ()
     ())

(main : IO Unit)
(main ≜ do
  (let (tree ≜ make-tree (ι 10_000)))
  (p1 ← spawn (print-tree tree))
  (send tree p1))

Requires termination check to be disabled 

;; recursion
(inc : Actor Int → Behaviour Int)
(inc actor ≜ specify ⧺ λ [n] do
  (let (n' ≜ n + 1))
  (print n')
  (send n' actor))

;; prints 0 1 2 3 4 ...
(main ≜ do
  (p ← rec inc)
  (send 0 p))

;; mutual-recursion
(inc : Actor Int → Behaviour Int)
(inc actor ≜ specify ⧺ λ [n] do
  (let (n' ≜ n + 1))
  (print n')
  (send n' actor))

(dec : Actor Int → Behaviour Int)
(dec actor ≜ specify ⧺ λ [n] do
  (let (n' ≜ n - 1))
  (print n')
  (send n' actor))

;; prints 1 0 1 0 1 0 ... 
(main ≜ do
  ((p1, p2) ← mutual inc dec)
  (send 0 p1))
